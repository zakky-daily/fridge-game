# 「逃走する冷蔵庫」ゲーム

## Project Overview

このプロジェクトは、健康に関するシステム提案をテーマにした 3D Web ゲーム「逃走する冷蔵庫」を制作する。
プレイヤーは深夜の家の中で、意思を持って逃げ出した冷蔵庫を追いかける。ゲームとしてはシンプルな鬼ごっこだが、「体を動かすこと」「消費カロリー」「疲労状態に応じた選択」を物語と UI に組み込み、健康促進システムとしての意味が伝わるようにする。

ゲームの雰囲気はコミカル寄りにする。太った人や痩せる表現を扱うが、体型をからかう方向にはしない。プレイヤーが少しずつ前向きに運動できるようになる、という健康支援の文脈を前面に出す。

## Core Concept

- TPS 視点の 3D アクションゲーム。
- 舞台は深夜の家。
- プレイヤーは冷蔵庫を追いかける側。
- 冷蔵庫は逃げる対象であり、コミカルに喋るキャラクター。
- 消費カロリーが UI に表示される。
- カロリーを消費するほど冷蔵庫の速度が落ち、捕まえやすくなる。
- プレイヤーの見た目はカロリー消費量に応じて少しずつ痩せていく。
- 道中のアイテムを拾うと一時的にブーストできる。
- 難易度選択は「今日一日がどうだったか」を聞く形にする。
- すごく疲れている選択肢ではゲームを開始せず、休息を肯定するエンドへ進む。

## Target Platform

- GitHub Pages から公開し、URL にアクセスするだけで遊べる形にする。
- 3D Web フロントエンドのみで完結させる。
- スマホと PC の両対応にする。
- サーバー処理、ログイン、DB、ランキングなどは原則実装しない。
- ビルド後の静的ファイルだけで動作する構成にする。
- 画像や 3D モデルなどのアセットは `public/` 以下に置くか、簡易的なプリミティブ形状で代用する。

## Recommended Tech Stack

基本方針として、GitHub Pages で静的ホスティングできる技術だけを使う。

- TypeScript
- Vite
- Three.js
- HTML / CSS
- GitHub Pages
- GitHub Actions または `gh-pages` によるデプロイ

追加ライブラリは最小限にする。ゲームの目的は高度なゲームエンジン構築ではなく、健康システム提案として遊べるプロトタイプを作ること。

Vite を使う場合、GitHub Pages のリポジトリ名に合わせて `vite.config.ts` の `base` を設定する。ユーザーサイト形式なら `/`、プロジェクトサイト形式なら `/<repository-name>/` を使う。

## Visual Direction

- 深夜の家らしく、暗めの室内、廊下、キッチン、リビングを表現する。
- 全体はややデフォルメされたコミカルな見た目にする。
- 冷蔵庫には必ず目と口を付け、単なる直方体ではなくキャラクターとして見えるようにする。
- 冷蔵庫の目や口は、余裕がある時、からかっている時、焦っている時で表情が少し変わるとよい。
- 冷蔵庫には足、または揺れるモーションを付けるとよい。
- プレイヤーはシンプルな人型モデルでよい。
- プレイヤーの体型変化は、極端すぎず、徐々に体の横幅が小さくなる程度でよい。
- UI は健康アプリ風にし、摂取カロリー、ブースト状態、冷蔵庫との距離を見せる。
- 摂取カロリー UI は円形メーターにする。
- 円形メーターは、最初は赤色寄りで表示し、ゲームが進んで健康的な行動が増えるにつれて徐々に緑色側へ変化させる。

## Game Flow

### 1. Home Scene

ホーム画面には以下を置く。

- タイトル: 逃走する冷蔵庫
- はじめから
- ムービーを見る
- 操作説明

「はじめから」を選択すると、最初のムービーを流してから難易度選択へ進む。
「ムービーを見る」を選択すると、ホームからいつでも最初のムービーを再生できる。

### 2. Opening Movie

最初のムービーは短く作る。

内容:

1. 深夜の家。
2. 一人の太った人が冷蔵庫へ歩いていく。
3. 主人公もノベルゲーム風の吹き出しで独り言を喋る。
4. 冷蔵庫の扉に手をかけようとする。
5. 冷蔵庫が突然喋る。
6. 主人公と冷蔵庫の会話をノベルゲーム風の吹き出しで表示する。
7. 冷蔵庫が少しからかうようなセリフを言ってから走って逃げていく。

会話は、画面下部またはキャラクター付近の吹き出し UI で進行する。クリック、タップ、または一定時間経過で次のセリフへ進むようにする。

主人公のセリフ例:

> 「今日も疲れた……ちょっとだけ何か食べようかな。」

> 「えっ、冷蔵庫が喋った！？」

冷蔵庫のセリフ例:

> 「ちょっと待った！その一口、本当に今必要？」

> 「ふふん、追いつけるものなら追いついてごらん。」

> 「食べたいなら、まずは少しだけ動いてみない？」

冷蔵庫は少しからかっているような性格にする。ただし、主人公の体型や疲労を強く否定する言い方にはしない。説教ではなく、軽い挑発とコミカルな掛け合いでゲーム開始につなげる。

### 3. Difficulty Selection

難易度選択は、通常の Easy / Normal / Hard ではなく、健康システムらしく「今日一日がどうだったか」を聞く。

表示例:

> 今日一日、どうだった？

選択肢:

- すごく疲れている
  - ゲームを開始しない。
  - 休息を肯定するエンドへ進む。
- 少し疲れている
  - やさしい難易度で開始する。
  - 冷蔵庫の初期速度を遅めにする。
  - ブーストアイテムを多めに置く。
- まだ動けそう
  - 通常難易度で開始する。
  - 冷蔵庫の初期速度を標準にする。
  - アイテム数は標準にする。

「すごく疲れている」を選んだ場合のエンドは、健康提案として重要な要素にする。
休むことも健康行動である、というメッセージを入れる。

### 4. Gameplay Scene

基本ルール:

- プレイヤーは TPS 視点で家の中を移動する。
- 冷蔵庫はプレイヤーから逃げる。
- プレイヤーが冷蔵庫に一定距離まで近づくと捕まえた判定になる。
- 捕まえたらリザルトへ進む。

ゲーム中の主な UI:

- 摂取カロリー
- 摂取カロリーを示す円形メーター
- 冷蔵庫との距離
- ブースト残り時間
- 現在の体調モード

カロリーの扱い:

- 移動している間、少しずつカロリーが増える。
- ダッシュやブースト中は増え方を少し大きくする。
- カロリーが一定値を超えるごとに、冷蔵庫の速度を段階的に遅くする。
- カロリーが増えるごとに、プレイヤーの体型を少しずつ変化させる。

UI 表示上は、単に「消費カロリー」と見せるよりも、深夜に食べようとしていた分の「摂取カロリー」を健康行動によって抑えていく、という見せ方にする。円形メーターは赤色から始まり、移動やアイテム取得によって徐々に黄色、黄緑、緑へ変化する。これにより、プレイヤーが動くほど健康的な状態に近づいていることを直感的に示す。

冷蔵庫の挙動:

- プレイヤーから一定距離を保つように逃げる。
- 壁や家具にぶつからないように、簡単なウェイポイント移動でよい。
- 高度な AI や複雑な経路探索は必須ではない。
- 時々、吹き出しで一言喋ると面白くなる。

冷蔵庫のセリフ例:

- 「まだ追ってくるの！？」
- 「その調子、意外といい運動だよ！」
- 「カロリーが燃える音がする……！」
- 「だんだん足が重くなってきた……冷蔵庫なのに！」

アイテム:

- 道中に健康的なアイテムを配置する。
- 例: 水、タオル、スニーカー、プロテイン風ドリンク、謎の健康エナジー。
- 取得すると一定時間だけ移動速度を上げる。
- アイテム取得時に少しだけ演出を入れる。

### 5. Result Scene

捕まえた後、リザルト画面を表示する。

表示内容:

- 消費カロリー
- 捕まえるまでの時間
- 拾ったアイテム数
- 体型変化の簡単な演出
- 健康メッセージ

健康メッセージ例:

- 「少し動くだけでも、体はちゃんと反応している。」
- 「今日の一歩が、明日の習慣になる。」
- 「無理せず続けることが一番大事。」

## Game Mechanics

### Player Movement

PC:

- WASD または矢印キーで移動。
- Shift でダッシュ。
- マウス移動またはドラッグでカメラ操作。

スマホ:

- 左下に仮想スティック。
- 右側ドラッグでカメラ操作。
- 右下にブーストボタン。

スマホでは画面スクロール、長押し選択、ダブルタップズームが起きないように CSS とイベント処理を調整する。

### Camera

- TPS 視点。
- プレイヤーの少し後ろ、少し上から見る。
- カメラはプレイヤーに追従する。
- 壁抜け対策は簡易でよい。プロトタイプではカメラ距離の固定でもよい。

### Calorie System

内部値として `calories` を持つ。

例:

- 歩行中: 1 秒あたり +0.8 kcal
- ダッシュ中: 1 秒あたり +1.5 kcal
- ブースト中: 1 秒あたり +2.0 kcal

実際の医学的な正確性よりも、ゲームとして分かりやすい増え方を優先する。ただし UI や文言では「推定」や「ゲーム内カロリー」と分かるようにする。

画面上の主表示は「摂取カロリー」とし、円形メーターで表現する。これは実際に食べた量ではなく、冷蔵庫を追いかけることで深夜の摂取衝動をどれだけ健康的な行動へ変換できたかを示すゲーム内指標にする。メーター色は以下のように変化させる。

- 序盤: 赤色。深夜に食べたい気持ちが強い状態。
- 中盤: 黄色から黄緑。体を動かして流れが変わってきた状態。
- 終盤: 緑色。健康的な行動に近づいた状態。

例:

```ts
const rate = clamp(calories / GAME_CONFIG.calories.maxForScaling, 0, 1);
const meterColor = lerpColor('#e53935', '#43a047', rate);
```

### Fridge Speed Scaling

冷蔵庫の速度は、消費カロリーに応じて下げる。

例:

```ts
fridgeSpeed = baseFridgeSpeed * clamp(1.0 - calories / 120, 0.45, 1.0);
```

これにより、プレイヤーが動き続けるほど冷蔵庫を捕まえやすくなる。
「運動した結果、ゲームが有利になる」という健康システムのテーマを明確にする。

### Player Body Shape Scaling

プレイヤーの体型変化は、モデル全体ではなく胴体の横幅を中心に変化させる。

例:

```ts
bodyScaleX = lerp(1.25, 0.85, clamp(calories / 100, 0, 1));
bodyScaleZ = lerp(1.25, 0.85, clamp(calories / 100, 0, 1));
```

変化は滑らかにし、極端な見た目にならないようにする。

### Catch Condition

プレイヤーと冷蔵庫の距離が一定以下になったら捕まえた扱いにする。

例:

```ts
if (distance(player.position, fridge.position) < 1.5) {
  finishGame();
}
```

## Health Theme Requirements

このゲームは単なる追いかけっこではなく、健康に関するシステム提案として見える必要がある。
以下を必ず意識する。

- 体調に応じてゲーム開始の可否や難易度が変わる。
- 疲れている人に無理に運動させない。
- 休息エンドを失敗扱いにしない。
- 消費カロリーを UI に出し、行動と健康指標を結びつける。
- 運動量がゲーム進行に影響する。
- 運動を罰ではなく、コミカルで前向きな体験として描く。

## Deployment Guidelines

- GitHub Pages で公開できるように、静的ファイルだけで動作させる。
- `npm run build` で `dist/` を生成する。
- `dist/` の中身を GitHub Pages にデプロイする。
- GitHub Actions を使う場合は、`main` ブランチへの push で自動デプロイされるようにする。
- 外部 API やサーバーが必要な機能は追加しない。
- アセットパスは相対パスまたは Vite の `base` 設定に従う形にし、GitHub Pages 上で 404 にならないようにする。
- スマホからもアクセスしやすいように、HTTPS の GitHub Pages URL で動作確認する。

## Implementation Guidelines

- まずはプリミティブ形状で実装する。
- プレイヤーは箱や球を組み合わせた人型でよい。
- 冷蔵庫は白い直方体に目と口を付けるだけでよい。余裕があれば表情差分を作る。
- 家は床、壁、簡単な家具で構成する。
- 1 ファイルが大きくなりすぎたら、責務ごとに分割する。
- ゲームループは `requestAnimationFrame` を使う。
- 時間依存の処理は `deltaTime` ベースで書く。
- スマホ操作と PC 操作を同じ `Input` クラスに集約する。
- UI は HTML/CSS で重ねる方針でよい。
- Three.js 側に UI を作り込みすぎない。

## Coding Style

- TypeScript を使い、主要な状態には型を付ける。
- 過剰な抽象化は避ける。
- 小さなプロトタイプとして読みやすいコードを優先する。
- コメントは意図が分かりにくい箇所にだけ書く。
- 仕様変更に備え、ゲーム定数はなるべく一箇所にまとめる。
- 既存コードを修正する場合は、差分を小さくする。

## Making the Game More Interesting While Keeping It Simple

ゲームは複雑にしすぎない。ただし、以下のような軽い工夫で面白さを増やす。

- 最初のムービーでは主人公と冷蔵庫の掛け合いを入れる。
- 冷蔵庫がときどきノベル風の吹き出しで喋る。
- カロリーが一定値を超えるたびに冷蔵庫が焦るセリフを出す。
- アイテム取得時に短い効果音や光る演出を入れる。
- 冷蔵庫が廊下、キッチン、リビングを逃げ回る。
- 途中で「誘惑ゾーン」を置き、近づくと少し減速する。
- 健康アイテムを取ると誘惑ゾーンの影響を打ち消せる。
- リザルトで「今日の健康タイプ」を表示する。

誘惑ゾーンの例:

- お菓子棚
- ソファ
- 深夜のテレビ

健康アイテムの例:

- 水
- タオル
- スニーカー
- ストレッチマット

これらは健康テーマを強めるための演出であり、実装が重くなる場合は省略してよい。

## Non-Goals

このプロジェクトでは、以下を目的にしない。

- 医学的に正確なカロリー計算。
- 本格的なダイエット指導。
- 体型への否定的な表現。
- 高度な 3D アセット制作。
- オンライン対戦。
- サーバー連携。

- 冷蔵庫を捕まえるとリザルトが表示される。
- 健康に関するシステム提案として意図が伝わる。

## Priority

実装の優先順位は以下の通り。

1. 遊べること。
2. 健康システムとしての意図が伝わること。
3. スマホと PC の両方で操作できること。
4. 冷蔵庫のキャラクター性が出ること。
5. 見た目を整えること。

見た目や演出よりも、まずは一連の体験が最後まで通ることを優先する。
